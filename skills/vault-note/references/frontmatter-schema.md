# Frontmatter Schema

Every note generated by `/vault-note` must start with YAML frontmatter following this schema.

## Required Fields

```yaml
---
title: "<note title>"
created: "YYYY-MM-DD HH:mm:ss"
source: "<session|research>"
source_type: "<conversation|web-research>"
tags:
  - <topic-tag>
  - <topic-tag>
---
```

## Optional Fields

```yaml
related:
  - "[[Related Note Title]]"
```

## Field Rules

- **Quote all scalar string values** in frontmatter by default. This avoids YAML parse issues with `:`, `#`, date-like values, and similar edge cases.
- `title`: Use the user-provided title when available. Otherwise infer a short descriptive title from the content. Always quote.
- `created`: Use the current local datetime at note creation time. Format: `"YYYY-MM-DD HH:mm:ss"`. Always quote.
- `source`: Use `"session"` for conversation-based notes, `"research"` for web-research-based notes. Always quote.
- `source_type`: Use `"conversation"` for session mode, `"web-research"` for research mode. Always quote.
- `tags`: Include 2-5 topic tags derived from the content. Use lowercase kebab-case (e.g., `python-decorators`, `web-development`).
- `related`: Include only when [[]] links to existing vault notes are found. Each entry must be a quoted wiki-link string.
- Omit unknown or unverifiable fields rather than guessing.
